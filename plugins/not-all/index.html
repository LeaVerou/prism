How to use

Not all provides you with ways to filter the element the highlightAll and highlightAllUnder methods actually highlight. This can be very useful when you use Prism's automatic highlighting when loading the page but want to exclude certain code blocks.

API

In Prism.plugins.notAll you can find the following:

filter(condition: (value: { element, language: string }) => boolean): void
Only if the given function return true will the element be highlighted. This can be used to define a custom language filter.
filterCss(selector: string): void
Only elements which match the given CSS selector will be highlighted.
excludeCss(selector: string): void
Only elements which do not match the given CSS selector will be highlighted.
filterKnown: boolean = false
Set this to true to only allow known languages. Code blocks without a set language or an unknown language will not be highlighted.

Attributes

You can also add the following data-* attributes to the script which contains the Not all plugin.

<script src="..." data-filter-css="<css selector>">
This attribute is a shorthand for Prism.plugins.notAll.filterCss. The value of the attribute will be passed as is to the filterCss function.
<script src="..." data-exclude-css="<css selector>">
This attribute is a shorthand for Prism.plugins.notAll.excludeCss. The value of the attribute will be passed as is to the excludeCss function.
<script src="..." data-exclude-known>
This attribute can be used to set the value of Prism.plugins.notAll.filterKnown. filterKnown will be set to true if the attribute is present, false otherwise.

Examples

The following code is used to define a filter on this page.

// <code> elements with a .no-highlight class will be ignored
Prism.plugins.notAll.excludeCss('code.no-highlight');
Prism.plugins.notAll.excludeCss('pre.no-highlight > code');

// don't highlight CSS code
Prism.plugins.notAll.filter(function (env) {
	return env.language !== 'css';
});

The results:

let foo = "I'm not being highlighted";
a.link::after {
	content: 'also not being highlighted';
	color: #F00;
}

Prism will ignore these blocks, so you can even define your own static highlighting which Prism would normally remove.

a.link::before {
	content: 'I just do my own highlighted';
	color: #F00;
}
