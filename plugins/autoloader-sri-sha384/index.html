Autoloader SRI (SHA384)

Adds subresource-integrity verification to the languages automatically loaded by the Autoloader.

How to use

Just select this plugin on the Download page and it will automatically enable SRI for all language definitions including minified ones.

You can disable SRI like this: 

Prism.plugins.autoloader.sri.enabled = false;

How it works

SRI works by computing a checksum of a downloaded resources and checking it against a list of valid checksums. Only if the computed checksum valid, will the resource be parsed and executed. More information.

This plugin uses the SHA384 hash algorithm to compute the checksums of language files. This algorithm was chosen because it present a good compromise between security and the file size of this plugin.

Examples

var a = 9;

The checksum of CSS was changed to a wrong hash causing CSS to fail to load. 

a {
	color: red;
}

Autoloader won't even attempt to load languages for which there isn't a checksum registered. This means unknown languages won't be loaded at all. 

a {
	color: red;
}
